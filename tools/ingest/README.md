# tools/ingest/ — 硬数据自动获取管线

> 目标：**全自动**从官方源获取硬数据建库，**不靠人工下载/转录**，且**数字绝不经过 LLM**（守 DESIGN 铁律一）。
> 这是**构建期工具**（build-time），与运行期 `lib/` 解耦。产物写入 `data/provinces/<省>/`，供 `lib/` 确定性查表。
>
> **现状（2026-06）**：广东首发已全自动落地（投档/一分一段/招生计划=文本PDF，选科要求=官方选考系统）。当前唯一需人工提供的是组级真过滤所需的《招生专业目录》——见 [人工下载清单.md](./人工下载清单.md)（非必需，仅升级用）。

---

## 0. 为什么不能简单"实时查询"

官方（教育部阳光高考 / 各省考试院）**没有公开 API**；数据以 PDF / HTML 表 / Excel 形式发布，URL 年年变、页面会断。硬数据一年只变一次（出分后即固定）。所以正确形态是：

> **每年高考季，对目标省做一次"官方数据快照 ingest"** —— 自动抓取官方原件 → 确定性解析 → 自动校验 → 落库版本化。不是常年维护的重型 DB，是**季节性快照**。

详见根 DESIGN「数据获取决策树」。

## 0.5 硬数据来源途径（共 7 条，落地必读）

> 实测梳理（2026-06）。前 6 条满足铁律一（官方/权威、可回溯），第 7 条是红线。
> 每条标：产出什么 / 真实格式 / 可机读性 / 获取难点 / 本项目定位。

| # | 途径 | 产出的硬数据 | 真实格式（实测）| 可机读 | 获取难点 | 项目定位 |
|---|---|---|---|---|---|---|
| **1** | **省考试院官网·公告附件** | 一分一段、投档线/最低位次、批次线 | **因省异**：广东=文本PDF直链、山东=Excel(.xls)、湖北=图片JPG | 视格式 | URL年变、页面断、格式杂、**需绕代理直连** | **位次/分数线 权威主源** |

**各数据集 ← 途径映射**：

| 数据集 | 主途径 | 备注 |
|---|---|---|
| 时间线 | 1（省考试院公告）| + 4 兜底 |
| 一分一段表 | 1 | 广东文本PDF / 山东xls / 湖北图片 |
| 投档单位最低位次 | 1 | 广东投档PDF**同时含计划数** |

## 1. 管线总览（6 层）

```
①源注册表 registry        声明式：每省每数据集的官方URL、格式、列映射、校验规则
        │
②抓取+归档 fetch           allowlist官方域 → 重试/镜像 → 原件落盘+sha256 → 记录fetched_at
        │   (原件永久归档：页面断了也能复算、可审计)
③确定性解析 parse          按格式(HTML/Excel/文本PDF)+列映射，按列索引取值 → 行
        │   ★ 数字只由解析器产出，LLM 绝不参与取数
④留痕标注 provenance       每行打 source_url / year / fetched_at / raw_hash / parser_version
        │
⑤自动校验闸门 validate      单调性/边界/跨集一致性/与原件可回溯；任一不过→拒绝快照
        │
⑥落库版本化 publish         通过才写 data/provinces/<省>/，标记 current good；失败→保留上个good
```

## 2. LLM 的边界（铁律一的工程保证）

LLM **只允许做"定位/映射"，输出结构描述，绝不输出数字**：

| ✅ 允许（产出结构，可审计） | ⛔ 禁止（产出/触碰数字） |
|---|---|
| 在官方域内**挑出正确的公告/附件 URL** | 从页面**读出**任何位次/分数/计划数 |
| 判断页面格式（HTML表/Excel/PDF） | **填补**缺失单元格 |
| 把易变表头**映射到 schema 字段**（→存进注册表的 `column_map`） | **修正/插值**任何数值 |
| 选出页面里哪张 `<table>` 是目标表 | **汇总/重述**数字 |

**机制**：LLM 的映射结果写进**注册表 `column_map`（人可复核的配置）**，一次设定、长期复用。之后 ingest 完全确定性——解析器按列索引取值，**LLM 产出的内容里永远不出现进库的数字**。这样"自动化"与"零幻觉"并存。

## 3. 源注册表（registry，声明式）

`tools/ingest/sources/<省>.json`，每个数据集一条：

```json
{
  "省份": "湖北",
  "datasets": [
    {
      "dataset": "一分一段表",
      "year": 2026,
      "科类": "物理类",
      "publisher": "湖北省教育考试院",
      "primary_url": "https://www.hbea.edu.cn/.....html",
      "mirror_urls": ["https://镜像/同一公告"],
      "format": "html_table",
      "table_selector": { "index": 0 },
      "column_map": { "分数": 0, "本段人数": 1, "累计人数": 2 },
      "discovery": { "query": "湖北 2026 物理类 一分一段表", "allow_domains": ["hbea.edu.cn"] }
    }
  ]
}
```

- `primary_url` / `mirror_urls`：官方源 + 镜像（考试院常同时发在教育厅/政务网）。
- `column_map`：表头→列索引（LLM 辅助生成、人复核、确定性复用）。
- `discovery`：URL 失效时，在 **allow_domains（仅官方域）** 内自动重新发现候选 URL（LLM 只选 URL，不取数）。

**域名 allowlist**（只认官方/权威）：`*.edu.cn`（考试院/高校）、`*.gov.cn`（教育厅/政务）、`chsi.com.cn`（阳光高考）。其余一律拒绝（含一切第三方聚合站）。

## 4. 抓取 + 归档（解决"页面会断"）

> ⚠ **2026-06 实测教训（非假设）**：抓真实官方源时发现——
> - 官方 `hbea.edu.cn` 公告页从境外**反复断连**（socket closed）；
> - 官方 `hbccks.cn` **http-only / TLS 证书域名无效**（`ERR_TLS_CERT_ALTNAME_INVALID`）；
> - **强制 http→https 升级会直接挂**。

- 仅抓 allowlist 域；**不强制升 https**；对 allowlist 内官方域**容忍坏证书**（原件落盘 + sha256 + 校验闸门把关，风险可控）；重试+退避；primary 失败转 mirror。
- 境外/CI 环境可能根本连不上考试院站 → 支持**"已归档原件复算"模式**（`allow_local`：喂入此前抓到的原件离线重解析），作为不可达兜底。
- **抓到立即把原件字节归档** 到 `raw/<省>/<dataset>-<year>-<sha256前12>.<ext>`，并记 `fetched_at`、页面自身发布日期（若有）。
- 原件归档是**审计与复算的根**：页面以后断了，仍可重解析、可向用户回溯出处。

## 5. 确定性解析（数字来源）

> ⚠ **2026-06 实测教训（推翻原假设）**：原以为"HTML 表是主路径、OCR 是边缘"——**错**，各省格式**不统一**。
> 实测**湖北 2025 一分一段表官方就是 6 张 JPG 图片**（连聚合站 eol.cn 也只能转载图片）；而**广东（首发）一分一段+投档表全是文本PDF**、山东是 Excel。
> 所以判据是**按省核验真实格式**：**文本PDF/Excel 能机读的省优先上线（免 OCR）**；**OCR 只对图片省（如湖北）才是核心路径**。
> 现状：**广东首发已走文本PDF 路径落地（`pdf_text` 已实现）**，OCR 待图片省上线时再接。落地前**必须对每个目标省先实地核验真实格式**。

按 `format` 分派，**全部按列索引/坐标取值，不做语义猜测**：

| format | 解析方式 | 依赖 | 状态 / 真实占比 |
|---|---|---|---|
| `pdf_text` | pypdf 逐页抽文字 → 按行正则取值 | pypdf | **✅ 已实现**（广东首发即走此路：一分一段+投档表全文本PDF）|
| `html_table` | stdlib HTMLParser 抽第 N 张表 | 无 | **✅ 已实现**（部分省份）|
| `excel` | openpyxl/xlrd 按列索引读 | openpyxl/xlrd | ⬜ 待接（山东 .xls 等 Excel 省再接）|
| `image` / `pdf_scan`（图片/扫描件）| 双 OCR 引擎交叉，逐位不一致→送人工复核队列 | OCR | ⬜ 待接（**仅图片省**如湖北一分一段才需，非首发必需）|

**OCR 不违反铁律一**（OCR 是确定性软件、不是 LLM，数字由软件产出），但 OCR **会错读**，故让数据"自检"成为强制项：

- **算术恒等校验**（已实现，见 §6）：`累计[s] = 累计[s+1] + 本段[s]`，OCR 错读任一数字都会破坏它 → 自动抓出。
- 双 OCR 引擎交叉，逐位不一致 → 送**人工复核队列**（只确认被标红的个位数单元格，**不是下载转录整表**）。这是无法 100% 去人工的残差，已最小化。
- 源优先级：能拿到 Excel/HTML/文本PDF 就优先（无 OCR 误差）；只有图片时才走 OCR + 上述自检。

## 6. 自动校验闸门（发布前必须全过）

数字进库前用**确定性规则**自检，任一不过 → 拒绝该快照、告警、保留上一个 good：

- **一分一段·单调性 + 算术恒等**：分数降序时 `累计人数` 单调非降；`本段人数 ≥ 0`；**`累计[s] = 累计[s+1] + 本段[s]`（相邻整数分）——OCR/录入错读任一数字都会破坏此恒等式，是图片源的"自检码"**；末行累计 ≈ 该科类报考总数（有官方总数则交叉核对）。
- **位次·边界**：`0 < 位次 ≤ 科类总人数`；`最低分` 落在合理分数域。
- **招生计划**：`计划数 > 0`；汇总与专刊总数交叉核对（若有）。
- **选科要求·枚举**：`首选 ∈ {物理,历史,不限}`、再选 ∈ 已知科目集。
- **跨数据集一致性**：每个投档单位在位次表/选科表/计划表都能对上；`最低分` 经一分一段能换算出与 `最低位次` 相符的值（容差内）。
- **可回溯**：每个进库单元格能指回原件 `raw_hash`。

抽检并未取消，而是**升级为自动规则**；规则覆盖不到的（如扫描件 OCR）仍按 DESIGN §8 随机抽样人工核对并记 `data/README.md`。

## 7. 留痕与版本化

- 每行强制 `source_url / year / fetched_at`（SCHEMAS 契约）+ 内部 `raw_hash / parser_version`。
- 快照**不可变、按 `省/年/版本` 版本化**；服务层只指向「current good」。
- ingest **幂等**：同一原件 sha256 未变 → 跳过；变了 → 新版本 + 重校验。

---

## 8. 季节性调度

- 由 `data/provinces/<省>/时间线.json` 的出分日驱动：临近出分开始**轮询注册表 URL** 探测"是否已发布"（页面是否出现目标表）。
- 探测到 → 自动 fetch→parse→validate→publish；失败重试并告警。
- 适合用 `/loop` 或 cron 在出分季对目标省轮询（见根 DESIGN §11 路线图）。

---

## 9. 覆盖兜底（绝不编造）

某省/某年快照不存在或校验未过 → **Q1 对该省优雅降级**：不产任何位次清单，转「方法论 + 指向官方阳光志愿/考试院查询入口」。**宁可不答，绝不编数字**（守铁律一、衔接根 DESIGN §3.3 与"覆盖兜底"）。
