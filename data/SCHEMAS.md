# data/ 数据契约（SCHEMAS）

> 本文件是 `data/` 下所有结构化文件的**字段契约**，是 `lib/` 确定性逻辑读取数据的唯一依据。
> 改字段须同时改本文件与 `lib/`。
>
> **强制公共字段（每条硬数据都必须有）**：`source_url`、`year`、`fetched_at`。
> 缺 `source_url`/`year` 的行视为脏数据，建库时丢弃。
>
> **来源留痕字段（点级可回溯，支持用户细究来源）**——每行还带：
> - `source_url`：官方文件直链（PDF/Excel/ZIP）
> - `公告页URL`：承载该文件的官方公告页（比直链更稳、有上下文）
> - `源文件名`：当 `source_url` 是压缩包时，包内对应文件名（直链可留空）
> - `发布机构`：如「广东省教育考试院」（用户友好展示）
> - `year`：数据年份　`fetched_at`：抓取日期
> - `raw_hash`：原件 sha256（= `data/_raw/<省>/` 归档原件哈希，可验真/离线复算）
> - `parser_version`：解析器版本（复算可追溯）

## 0. 投档单位粒度（贯穿 Q1 所有文件）

Q1 的数据粒度 = 该省**投档单位**，随省志愿模式变（见 DESIGN §4 第三波）：

| 省志愿模式 | 投档单位 | `投档单位id` 形态 |
|---|---|---|
| 院校专业组（湖北/江苏/北京…） | 院校专业组 | `<院校代码>-<专业组代码>`，如 `C001-03` |
| 专业(类)+院校（山东/浙江…） | 专业(类) | `<院校代码>-<专业(类)代码>`，如山东 `A001-17`（A001=山东自有院校代号，17=专业代号） |
| 传统文理（未改革省） | 院校 | `<院校代码>` |

**铁律**：新高考省的 `投档单位id` 必须落到组/专业类，**不得只到院校代码**（否则同校不同组位次被错误聚合，见 DESIGN P0 缺陷2）。

> **山东实测要点（2026-06-07 核验，已落地）**：山东用**自有院校代号**（A001…Y030，**非国标代码**），投档单位=**专业(类)**（每行一个专业/专业类，粒度比专业组更细）。一份《普通类常规批投档情况表》**同时给齐 计划数 + 最低位次**（只给位次、不给最低分）。详见 DESIGN §4 山东条目。

## 1. `provinces/<省>/时间线.json`（P0 阶段判定用）

```json
{
  "省份": "湖北",
  "省级行政区类型": "省",
  "试卷类型": "新高考I卷",
  "志愿模式": "院校专业组",
  "投档单位粒度": "院校专业组",
  "科类体系": "3+1+2",
  "可填志愿数": { "本科批": 45 },
  "年份": 2026,
  "考试": { "start": "2026-06-07", "end": "2026-06-09" },
  "出分日": "2026-06-25",
  "批次": [
    {
      "批次": "本科提前批",
      "覆盖": false,
      "填报start": "2026-06-26",
      "填报end": "2026-06-27"
    },
    {
      "批次": "本科批",
      "覆盖": true,
      "填报start": "2026-06-29",
      "填报end": "2026-07-02"
    }
  ],
  "source_url": "https://……（省考试院当年安排）",
  "fetched_at": "2026-06-05"
}
```

字段说明：
- `投档单位粒度`：必须与 §0 表一致；lib 据此校验 Q1 数据粒度。
- `科类体系`：`3+1+2`（广东/湖北等→按首选分 **物理类/历史类** 双轨，一分一段与投档都分轨）/ `3+3`（山东/浙江/沪→全省**综合**一个位次序列，**不分物理/历史**，选考3科只用于专业选科资格过滤）/ `文理`（未改革省）。`lib/answer_q1.py` 据此决定 `科类`：3+3 → `综合`；3+1+2 → 物理类/历史类。
- `可填志愿数.<批次>`：用于 §5 梯度配额建议（冲/稳/保各填几个）。
- `批次[].覆盖`：`true`=本 skill 覆盖（仅普通批平行志愿）；`false`=不覆盖（提前批等，DESIGN §1）。lib 阶段判定只对 `覆盖=true` 的批次产候选清单。
- 日期一律 `YYYY-MM-DD`。所有日期口径见 §3 阶段判定逻辑。

## 2. `provinces/<省>/一分一段表.csv`（分数↔位次换算，Q1 用）

| 列 | 类型 | 含义 |
|---|---|---|
| `科类` | str | 物理类 / 历史类（3+1+2 新高考）；**综合**（3+3 省如山东，全省一个综合位次序列，取一分一段『全体』列）；理科 / 文科（传统） |
| `分数` | int | 该分数段的分数 |
| `本段人数` | int | 恰为该分数的人数（可为 0） |
| `累计人数` | int | **≥该分数的总人数 = 该分数对应的位次** |
| `year` | int | 数据年份 |
| `source_url` | str | 官方一分一段表链接 |
| `fetched_at` | str | 抓取日期 YYYY-MM-DD |

换算规则（lib `rank.py`）：考生位次 = 其分数所在行的 `累计人数`。分数缺行时取**就近且不高于**该分数的最近一行（保守，宁可位次略大）。一省一科类一年一份。

## 3. `provinces/<省>/选科要求.csv`（Q1 第一硬过滤器，新高考省必备）

> 这是 Q1 的**第一过滤器**（DESIGN §5 步骤1）：选科不匹配 = 没资格填，先于位次计算剔除。错一条即误导报考资格，抽检比例 10 条。

| 列 | 类型 | 含义 |
|---|---|---|
| `投档单位id` | str | 见 §0 |
| `院校名` | str | 院校全称 |
| `专业组代码` | str | 院校专业组省填组代码；专业类省填专业类代码；传统省留空 |
| `科类` | str | 物理类 / 历史类 / 不限（传统省填理科/文科） |
| `首选` | enum | `物理` / `历史` / `不限`（3+1+2 的首选科目要求） |
| `再选必选` | str | 必须**全选**的再选科目，逗号分隔；无则留空。例：`化学` 或 `化学,生物` |
| `再选可选` | str | **至少选一门**的再选科目，逗号分隔；无则留空。例：`化学,生物`（化学或生物任一即可） |
| `year` | int | 选科要求适用年份 |
| `source_url` | str | 官方选科要求对照表链接 |
| `fetched_at` | str | 抓取日期 |

匹配语义（lib `subject_filter.py`，三条全过才保留）：
1. `首选 == 不限` 或 `首选 == 考生首选科目`；
2. `再选必选` 中每一门都在考生再选集合内；
3. `再选可选` 非空时，其中至少一门在考生再选集合内。

> 2024 起最常见的最严格形态是「物理+化学双必选」→ `首选=物理, 再选必选=化学`。

#### 3+3 省（山东）选科要求.csv —— 投档单位级真过滤

山东 3+3 无首选/再选之分，考生 6 科任选 3。选科要求来自官方《公布普通高校拟在山东招生专业(类)选考科目要求的公告》(文本PDF)，经 `run_shandong_xuanke.py` 按 (院校名+基础专业名) join 到投档单位（实测覆盖 89%；未匹配者**不入库**→answer_q1 退回「待核对」，绝不臆造）。列：

| 列 | 类型 | 含义 |
|---|---|---|
| `投档单位id` | str | 见 §0（山东 `院校代号-专业代号`） |
| `院校名`/`专业组代码`/`专业名` | str | 同 §4 |
| `科类` | str | `综合` |
| `要求类型` | enum | `不限` / `均须`(列出科目须全选) / `任选`(列出科目选其一) |
| `要求科目` | str | 逗号分隔，规范为 {物理,化学,生物,政治,历史,地理}；`不限` 时空 |
| `选考要求原文` | str | 官方原文，如 `物理,化学(2门科目考生均须选考方可报考)`（展示用，可回溯） |
| `source_url`/`公告页URL`/`year`/… | | 留痕（同 §4） |

匹配语义（lib `subject_filter.sd_match`）：`不限`→通过；`均须`→要求科目全 ⊆ 考生3科；`任选`→交集非空。无资格 → 投档单位级 hard-exclude（真过滤·路线a，DESIGN §5 步骤1a）。

**可选静态字段**（用于 §5 城市/学费/民办过滤，缺失则不据此过滤）：
- `城市`：院校所在地级市（城市偏好过滤用）
- `办学性质`：`公办` / `民办` / `中外合作`（民办接受度过滤用）
- `学费`：年学费（int，元）

## 4. `provinces/<省>/投档单位最低位次.csv`（Q1 分档基准，近3年）

> **每个投档单位至少存近3年**（每年一行，用 `year` 区分）。lib 取均值分档、区间展示。
> **不要只存单年**——单年最低位次=压线捡漏离群值且有大小年波动（DESIGN §5 步骤3、§8）。

| 列 | 类型 | 含义 |
|---|---|---|
| `投档单位id` | str | 见 §0 |
| `院校名` | str | 院校全称 |
| `专业组代码` | str | 同 §3 规则（专业(类)平行省存**专业代号**，如山东 `17`/`E1`） |
| `专业名` | str | **可选**；专业(类)平行志愿省（山东等）存投档专业(类)名称，用于候选展示『院校·专业名』。院校专业组省可留空。 |
| `科类` | str | 物理类 / 历史类 / **综合**（3+3省）/ … |
| `year` | int | 录取年份 |
| `最低分` | int | 该年该单位投档最低分；**部分省（山东）官方只公布位次、不公布分→留空**（lib/校验闸门均容忍空最低分） |
| `最低位次` | int | 该年该单位投档最低位次（分档用的原始量；山东=**综合位次**） |
| `source_url` | str | 官方投档线/位次表链接 |
| `fetched_at` | str | 抓取日期 |

聚合（lib `chongwenbao.py`）：
- `r_avg` = 近3年 `最低位次` 均值（分档基准）；
- `r_band` = 近3年 `最低位次` 的 (min, max)（展示波动）；
- 单年最低位次仅作「乐观边界」参考，不作分档基准。

## 编码与通用约定

- 文件编码 UTF-8，CSV 含表头行，逗号分隔；字段内若含逗号用双引号包裹。
- 日期一律 `YYYY-MM-DD`。
- 缺失值留空字符串，不用 `NA`/`null`。
- 真实数据 `source_url` 必须是可回溯的官方/权威链接；自测样例用 `SAMPLE`（见 `lib/_selfcheck_data/`）以示区分。
