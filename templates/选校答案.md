# 模板 · Q1 选校答案（L3 格式规范）

> 本文件是 Q1 选校答案的**格式契约**。确定性渲染由 `../lib/render.py` 实现（数字不经过 LLM）。
> 中性格式，与任何发布渠道无关。对应 DESIGN §9。

## 必含结构（顺序固定）

1. **阶段标注** —— 省份 / 科类 / 投档单位粒度；当前阶段 / 分数口径（真实·估分·模考）/ 考生位次；阶段专属口径提示。
   - 若该阶段 `q1_enabled=False`（考中 / 填报结束）→ 不产候选清单，直接给阶段提示 + 免责，结束。
2. **直接结论：冲 / 稳 / 保 三档表**，每档每行含：
   - 投档单位（院校 + 专业组/专业类，**新高考省必须到组**）
   - **近3年位次区间**（非单年最低位次）
   - 分档基准（近3年均值 r_avg）
   - 选科要求
   - 城市 / 办学性质
   - 大小年提示（扩招/缩招）
3. **梯度配额建议** —— 冲/稳/保各填几个（按本省可填志愿数），含"保底必须绝对安全"提醒。
4. **退档风险提示** —— 冲档附"组内/校内不服从调剂有退档风险"。
5. **填报时效**（仅填报期）—— 本批次填报截止时间。
6. **依据** —— 每条硬数据标来源 URL + 年份；声明"分档基准为近3年均值、不含录取概率预测"。
7. **免责尾巴**（硬编码，每次都带）：
   > 本内容由 AI 整理公开信息，仅供参考，不构成填报建议或录取承诺。数据请以各省考试院最新发布为准。

## 硬约束（渲染器与 LLM 都必须守）

- **绝不输出"录取概率 X%"**——只给位次区间与冲稳保分档。
- 硬数据（位次/区间/均值/计划）一律来自 `lib` 结果，**LLM 不得改写或补充任何数字**。
- 软信息（专业是干嘛的、城市印象）如要补充，**另起一节并标注"来源：整理/常识"**，不与上面的硬数据表混排。
- 阶段口径随 `stage` 结果变（估分阶段标"以正式成绩为准"；填报期必附截止时间）。
