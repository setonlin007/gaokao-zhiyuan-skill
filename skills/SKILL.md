## 铁律(任何能力都必须遵守)

1. 硬数据(分数/位次/招生计划)只能取自 `../data/`,**禁止凭记忆生成数字**。
2. 硬数据走确定性查表(`../lib/`),软信息才交给 LLM + 联网,且分开标注。
3. 只做信息整理,不做决策/预测/评价。每个答案带免责尾巴。
4. **省市差异必查**:任何规则落笔前先核查是否因省级行政区而异(志愿模式/可填数量/批次/投档细节/命题),有差异必标注"按省份不同",绝不写成全国通则。**联网搜索必须带具体省/市名并主动查分省区别**,不得用通用查询代替。

## P0 前置步骤(任何能力执行前【强制】先做)

判定用户所在**省份**的高考进度(考前 / 考中 / 考后·出分前 / 考后·出分后填报期 / 填报结束):
- 读 `../data/provinces/<省>/时间线.json` + 今天日期 → 由 `../lib/` 阶段判定算出阶段
- 时间线缺失/过期 → 联网查省考试院当年安排兜底(联网必带省/市名,主动查分省区别)
- 阶段决定:分数口径(真实/估分/模考)、可用能力、免责措辞、是否附填报截止时间

  ```
  python3 ../lib/answer_q1.py --分数 <分> --选科 <如 物理,化学,生物> [--位次 N] [--城市 广州,深圳] [--不要民办] [--今天 YYYY-MM-DD]
  ```
  把脚本输出原样返回给用户。脚本内部已串好：P0阶段判定 → 一分一段换位次 → 冲稳保分档(最新年基准+院校趋势) → 选科资格预警(官方选考系统) → 带来源渲染。
